# Creating a Harness

A harness starts with the work the user is already doing manually.

Do not start with tools.

Start with the loop.

## Questions

1. What are you trying to accomplish?
2. What are you doing manually today?
3. What tools are involved?
4. What should the harness produce?
5. What should the harness not do?
6. What needs human approval?
7. What does success look like?
8. What does failure look like?
9. What should be audited?
10. How should the harness improve over time?

## Minimum viable harness

A minimum viable harness has:

- A clear objective
- Ordered stages
- One agent prompt per agentic stage
- Defined inputs and outputs
- Saved artifacts
- Audit criteria
- A definition of done

## Bad harness signs

- No artifacts
- Vague roles
- No approval gates
- No audit criteria
- Too many agents
- Hidden human work
- The workflow depends on unstated judgment
